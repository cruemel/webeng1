Checkliste für Webentwickler

Bewährte Praxis

  • Interne und externe Links überprüfen.
  • Den Inhalt auf Rechtschreibfehler überprüfen.
  • Die Website in allen Browsern testen.
  • Die Subdomain "www" entfernen.

Mobile Endgeräte

  • MobileOK Wert > 75
  • Den Metatag "viewport" einfügen.
  • Die korrekten <input> types verwenden.
  • Die Seite manuell in verschiedenen Emulatoren (oder auf den richtigen Geräten) testen.

Analyse

  • Verfügbarkeit überprüfen.
  • Beobachten der Websitenutzung.

Geschwindigkeit

  • Google PageSpeed Wert > 90
  • Yahoo YSlow Wert > 65
  • Http-Header überprüfen.
  • Bilder optimieren.

Benutzerfreundlichkeit

  • HTML5 Check
  • eigene Fehlerseite(n)
  • Favicon
  • Benutzerfreundliche URLs

optional:

  • Suchfunktion hinzufügen.
  • Anpassen an die Umgebung (ähnliche Webseiten?).

Semantik

  • Microdaten hinzufügen.
  • Semantik überprüfen.

SEO

  • SenSEO Wert > 85
  • Google Rich Snippets (?)
  • robots.txt
  • XML Sitemap einbauen.

Code Qualität

  • HTML überprüfen
  • CSS überprüfen
  • CSS Lint
  • JS Lint / JS Hint
  • Wie sieht es mit der Mehrsprachigkeit aus?
  • Automatische Tests

Barrierefreiheit

  • Den Farbkontrast überprüfen.
  • Test der WAI-ARIA Konformität.

Sicherheit

  • Cross-site Scripting
  • Cross-site request forgery

optional:

  • SSL
  • HSTS (HTTP over TLS/SSL)

Social Media

  • OpenGraph Protokol
  • Twitter Cards (?)
  • Facebbok Insights (?)
