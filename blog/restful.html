Rest = REpresentational State Transfer

Bestandteile

  1. Quelle

    • Dinge vs. Aktionen
    • Nomen vs. Verben
    • Rest vs. Soap

    Wobei die Quelle anhand der Uri identifiziert wird. Was widerum bedeutet: entscheidend ist nicht die Quelle, sondern deren Vertretung ...

  2. Vertretung

    • wie die Quelle(n) verändet werden (der body)
    • Teil des Zustandes der Quelle (der Transfer zwischen Client und Server)
    • typischerweise JSON oder XML
    • Beispiel:
      • Quelle: eine Person
      • Service: die Kontaktdaten (GET)
      • Vertretung: {name, address, phone} oder <name></name>

Bedingungen

  1. Einheitliche Schnittstelle

    definiert die Schnittstelle zwischen Client und Server

    • HTTP Aktionen (GET, PUT, POST, DELETE)
    • URIs (resource name)
    • HTTP Antwort (status, body)

  2. Verbindungslos

    • der Server hat keine Information über den Zustand des Client
    • jede Nachricht ist selbsterklärend
    • Sessions werden allein auf dem Client gespeichert

  3. Austausch zwischen Client und Server

    • Client und Server sind nicht miteinander verbunden
    • eine Trennung von Interface und Services
    • http als Verbindung zwischen Client und Server

  4. Pufferbar

    • bedingungslos
    • ausdrücklich
    • verhandelbar

  5. Verschiedene Anwendungsschichten

    • der Client kann keine direkte Verbindung zum Server erwarten
    • Vermittler zwischen Software und Hardware bzw. Client und Server

  6. Zusätzliche Skripte auf Anforderung

    • die Möglichkeiten des Clients werden erweitert, indem ein Teil der Logik zum Client verlagert wird
    • zum Beispiel: javascript
